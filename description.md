Платформа для студентов, помогающая находить участников для учебных/стартап-проектов. Пользователь создаёт проект (описание, что реализовано, какие цели), внутри проекта создаёт одну или несколько вакансий/позиций (например: фронтенд-разработчик) с требованиями по навыкам. Другие пользователи заполняют профиль и список навыков — им показывается персональная лента проектов и вакансий. Пользователь может откликнуться на позицию и стать участником (роль), а принять участника может владелец проекта или другие участники, у которых есть право «модерация/приём». Владелец при создании позиции получает рекомендации подходящих кандидатов (те, у кого профиль открыт). На старте вводятся базовые ограничения (сколько проектов/позиций может создать владелец, кто и сколько кандидатов может просматривать), а позже эти лимиты расширяются через подписку.

Функциональные требования

Регистрация / аутентификация (email/пароль, OAuth опционально).

Профиль пользователя:

ФИО/никнейм, контакт, краткое резюме.

Список навыков (теги), опыт, ссылки (GitHub, портфолио).

Статус видимости профиля: open / closed.

Проекты:

CRUD проектов (создать/править/удалить/просмотреть).

Поля: название, описание, статус (active/archived), что уже реализовано, цели, теги/навыки, публичность (public/private).

Позиции (в рамках проекта):

CRUD позиций: название позиции, описание, требуемые навыки, обязанности, роль по умолчанию, удалённость/локальность, ожидаемая загрузка.

Лента/фид:

Персонализованная лента вакансий/проектов по навыкам, тегам и поведению.

Фильтры (по навыкам, типу, удалённости).

Отклики и приглашения:

Пользователь может откликнуться на позицию (заявка).

Владелец/модератор проекта может приглашать пользователей (invite).

Рекомендации кандидатов для создателя позиции (по совпадению навыков, открытости профиля).

Участничество:

При успешном отклике пользователь становится member проекта с ролью (member, dev, reviewer и т. п.).

Решение принимается владельцем или участником с правом approve_members.

Права и ограничения:

Ролевые права: owner, admin_project, member, guest.

Базовые ограничения на просмотр кандидатов и количество создаваемых проектов/позиций (см. ниже).

Подписка / монетизация (фича-флаг):

Подписки, которые повышают лимиты (проекты/позиции/доступ к расширенным рекомендациям).

Уведомления:

Email / in-app / push (webhooks) о новых откликах, приглашениях, принятых/отклонённых заявках.

Логирование, мониторинг, метрики (через Prometheus/Grafana, Sentry).

API документация (OpenAPI/Swagger).

Тестирование: unit, интеграционные тесты, нагрузочные тесты.

Нефункциональные требования (highload, масштабируемость)

Должна выдерживать высокий трафик: миллионы пользователей и сотни тысяч запросов/мин при росте.

Горизонтальное масштабирование API (stateless приложения), использование кэша (Redis) для горячих данных и rate-limit.

Асинхронная обработка тяжёлых задач (рекоммендации, рассылки) через worker (Celery/RQ).

Поисковая система для быстрого поиска по навыкам/тегам/описаниям (Elasticsearch / OpenSearch).

Кеширование рекомендаций/фида (Redis) и TTL/инвалидация.

Оптимизация запросов к БД, шардирование/репликация PostgreSQL при необходимости.

Очереди и backpressure: обработка пиков через очередь задач и autoscaling.

Базовые ограничения (по умолчанию, конфигурируемые)

(Привожу разумные стартовые лимиты — их можно менять в конфиге.)

Бесплатный пользователь:

Максимум проектов: 3

Максимум позиций на проект: 5

Просмотров полного резюме кандидата в неделю: 20

Подписка (стандарт): проекты: 20, позиции на проект: 20, просмотров резюме/неделю: 200

Подписка (pro): без ограничений (или очень высокие лимиты)

Ограничения реализовать на уровне API (rate limit) и UI (скрыть кнопки), а также отчётность в админке.
